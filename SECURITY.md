# Security Policy

## Supported Versions

我们定期发布版本更新，包括安全修复。以下是目前支持的版本：

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## 报告安全漏洞

如果您发现安全漏洞或有任何安全相关的问题，请通过以下方式报告：

**请不要在 GitHub Issues 中公开报告安全漏洞！**

### 报告方式

1. **首选**: 发送邮件至项目的安全团队邮箱
2. **备选**: 通过 GitHub Security Advisories 报告

### 报告内容

请包含以下信息：

- 安全漏洞的类型和描述
- 复现步骤
- 受影响的版本范围
- 可能的修复方案（可选）

## 安全更新流程

1. 收到安全报告后，我们会尽快确认
2. 评估漏洞的影响和严重程度
3. 开发修复补丁
4. 发布安全公告和更新版本
5. 在安全公告中公开漏洞信息

## 安全特性

Senix Gateway 包含以下安全特性：

- **JWT 认证**: 基于 JWT 的 API 认证
- **密码加密**: 使用 bcrypt 进行密码哈希
- **SSL/TLS**: 自动 Let's Encrypt 证书管理
- **WAF**: 集成 Coraza WAF（开发中）
- **输入验证**: 严格的请求参数验证

## 依赖项安全

我们定期更新依赖项以确保安全。使用以下命令检查依赖项漏洞：

```bash
# Go 依赖检查
go audit

# 前端依赖检查
cd web && npm audit
```

## 配置安全建议

1. **修改默认凭证**: 首次部署后立即修改默认管理员密码
2. **使用 HTTPS**: 生产环境务必启用 HTTPS
3. **限制访问**: 配置防火墙规则限制管理端口访问
4. **定期更新**: 保持 Senix 版本为最新
5. **日志监控**: 定期检查日志以发现异常行为
